The ``server list`` argument is used to find instances that are associated with an |hp cloud| account. The results may show instances that are not currently managed by the |chef server|.

This argument has the following syntax::

   knife hp server list

This argument has the following options:

``-A ID``, ``--hp-account ID``
   |hp-account|

``-c CONFIG``, ``--config CONFIG``
   |config|

``--color``
   |color|

``--defaults``
   |defaults|

``-e EDITOR``, ``--editor EDITOR``
   |editor|

``-E ENVIRONMENT``, ``--environment ENVIRONMENT``
   |environment|

``-F FORMAT``, ``--format FORMAT``
   |format|

``-h``, ``--help``
   |help|

``--hp-auth URI``
   |hp-auth|

``-k KEY``, ``--key KEY``
   |key|

``-K SECRET``, ``--hp-secret SECRET``
   |hp-secret|

``--no-color``
   |no-color|

``-n``, ``--disable-editing``
   |no-editor|

``--print-after``
   |print-after|

``-s URL``, ``--server-url URL``
   |server-url|

``-T ID``, ``--hp-tenant ID``
   |hp-tenant|

``-u USER``, ``--user USER``
   |user|

``-v``, ``--version``
   |version chef|

``-V``, ``--verbose``
   |verbose|

``-y``, ``--yes``
   |yes|

``-Z ZONE``, ``--hp-zone ZONE``
   |hp-zone|
